# Workflow B：数字孪生与战略推演（商业 / 咨询）

本文描述 **商业 / 咨询** 场景下，现实数字化、反事实推演与决策情报如何与 [Core Engine](core-egnine.md) 衔接。引擎的**步级迭代**、**仿真级迭代**（蒙特卡洛）与**外部干预**见 [engine/core-engine.md](../engine/core-engine.md) §1、§5.1、§5.2。面向产品与商务侧设计讨论；不展开 L0–L6 的数据结构与具体 Schema。

---

## 1. 场景定义

| 项目 | 说明 |
|------|------|
| **目标客户** | 投行、律所、企业战略部、高端行为学研究机构等 |
| **核心价值** | 「复盘与预演」：把过往谈判变为数据，把未来决策变为可重复实验 |
| **关键词** | High Fidelity, Cloning, Counterfactuals, Deep Sensing |

---

## 2. 数据流与引擎衔接

### 2.1 Phase 1：现实数字化（Digitization）

**Standard 输入**：真实谈判**录音**（Audio）或**会议纪要**（Transcript）。

**过程（L0–L1）**：ASR 转写、说话人分离；从文本或事件流中抽取**关键事实**（如初始报价、核心关切等），供后续状态初始化与人格拟合使用。

#### 分支：深度感知接入（Research / Pro）

| 项目 | 说明 |
|------|------|
| **定位** | 高端咨询 / 心理学研究的 **近似研究工具**（Research Approximation Tool） |
| **输入** | 生理信号（EEG / EDA / HRV）或面部微表情等，经 Riff / 摄像头 / 可穿戴设备采集 |
| **过程** | **Signal Alignment**：将生理波峰（如应激峰值）与谈判时间线毫秒级对齐；**Hidden State Extraction**：挖掘「言行不一」时刻（如口头同意但 HRV 显示抗拒），为人格拟合提供**潜意识参数** |

#### L3 人格拟合（Persona Fitting）

**输入**：历史数据——**仅文本**（转写 + 事件）或 **文本 + 感知数据**。

**过程**：反推对手的**人格与策略参数**（如冲动度、应激阈值、掩饰倾向等）。仅文本时得到行为层参数；加入感知数据时可进一步估计心理层面参数。

**输出**：与真实对手高度近似的 **AI Clone**（Profile + 可选潜意识参数），用作 Core Engine 的 Agent Profile，支撑高保真反事实推演。

### 2.2 Phase 2：调用核心引擎（Simulation）

**用户设定**：反事实策略或干预，例如「若当时不接受 54.20、而坚持 50 会怎样？」。

**引擎调用**：传入 **Profiles**（如 Fitted_Musk_Clone_Advanced）、**Initial State**（Real_World_State_T0）、**User Intervention**（如 Offer_50）；可选 **Simulation Mode**（如 DEEP_PSYCH，开启心理动力学相关规则或参数）。

**执行**：引擎进行**仿真级迭代**（如 100 次蒙特卡洛）。每次 run 从初始状态 + 用户干预出发，完成整条步级迭代直至终止；聚合多条 Trace 与 Outcome，得到破裂概率、对方非理性崩溃点等统计结论。多 run 可由引擎内置或由上层多次调用单次仿真，见 [engine §5.2](../engine/core-engine.md)。

### 2.3 Phase 3：决策情报（Intelligence）

**输出（L6）**：**Risk & Opportunity Report**。

| 类型 | 说明 |
|------|------|
| **标准结论** | 基于多条推演聚合的结论，如「在 100 次推演中，坚持报价 50 导致谈判破裂的概率为 65%」 |
| **感知增强结论**（Research 分支） | 结合生理/微表情等信号，指出对手在特定议题下的认知负荷、薄弱时刻，并给出策略建议（如在该时点引入复杂条款） |

## 3. 商业特性

| 特性 | 说明 |
|------|------|
| **分层服务** | **Standard**：仅需录音，面向多数商务复盘；**Research / Pro**：配合感知设备，提供近似「测谎级」的心理博弈分析 |
| **数据隐私** | 私有化部署；生物识别与生理数据严格加密，不出本地 |
| **高精度模版** | 针对 M&A、劳资纠纷等场景，定制不同的 **CausalTransition** 参数集，与 [engine](../engine/core-engine.md) 转移层域参数一致 |
